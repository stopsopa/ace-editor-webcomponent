Lesson 001: Basic Custom Element

What You'll Learn

  • How to define a custom element
  • Basic lifecycle: connectedCallback()
  • Custom element naming requirements
  • Extending HTMLElement

Key Concepts

A custom element must:

  1. Have a hyphen in its name (e.g., my-element)
  2. Extend HTMLElement
  3. Be registered with customElements.define()

Live Example

Output:

The Code

<hello-world></hello-world>

<script>
class HelloWorld extends HTMLElement {
    connectedCallback() {
        this.innerHTML = `
            <div>
                <h3>Hello from a Web Component!</h3>
                <p>This is a basic custom element.</p>
                <p>Created at: ${new Date().toLocaleTimeString()}</p>
            </div>
        `;
    }
}

customElements.define('hello-world', HelloWorld);
</script>

How It Works

  1. Class Definition: We create a class that extends HTMLElement
  2. connectedCallback(): This lifecycle method runs when the element is added to the DOM
  3. Registration: customElements.define() registers our element
  4. Usage: We can now use <hello-world> like any HTML element

Try It Yourself

Open the browser console and type:

const elem = document.createElement('hello-world');
document.body.appendChild(elem);

Key Takeaways

  • Custom elements let you create reusable components
  • connectedCallback() is called when the element enters the DOM
  • The element name must contain a hyphen
  • No Shadow DOM or templates yet - just basic functionality
