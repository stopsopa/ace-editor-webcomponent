aceOnLoad Event

Before we dive into dynamic controls, let me introduce a powerful global event that's available for advanced configuration scenarios. The 'aceOnLoad' event is not always needed for basic usage, but it becomes invaluable when you need to configure each editor instance beyond the standard attributes available on the main <ace-editor> tag.

Use it to apply advanced configurations, attach custom event handlers, or integrate with other libraries that need direct editor access.

Auto-Generated IDs & Duplicate Detection

    

      üÜî Understanding Editor IDs: Each <ace-editor> component requires a unique ID
      to function properly. The component handles IDs in two ways:
    


    

      
Example 1: Auto-Generated IDs (Recommended)

      

        When you don't provide an id attribute, the component automatically generates a
        unique ID like ace-editor-1, ace-editor-2, etc. This is the
        easiest and safest approach for most cases.
      


      

        

          
Without ID (auto-generated)


          
<ace-editor lang="javascript">
  console.log("Hello, World!");
</ace-editor>


          
          
        

      


      
        Don't provide an ID unless you specifically need to reference the editor by ID. Let the component auto-generate
        IDs to avoid conflicts.
      

      
Example 2: Manual IDs (Use with Caution)

      

        When you do provide an id attribute, you are responsible for ensuring uniqueness.
        If you accidentally use the same ID twice, the component will throw an error.
      


      

        

          
‚úÖ With Unique ID (valid)

           console.log("Hello, World!"); 
          
<ace-editor id="my-editor-1" lang="javascript">
  console.log("Hello, World!");
</ace-editor>

        


        

          
‚úÖ With Different ID (valid)

           print("Hello, World!") 
          
<ace-editor id="my-editor-2" lang="python">
  print("Hello, World!")
</ace-editor>

        

      


      
        When providing manual IDs, you must ensure each ID is unique across
        all ace-editor components on the page. Duplicate IDs will cause errors!
      

      
Example 3: Duplicate ID Error

      

        ‚ùå The second editor below intentionally uses a DUPLICATE
        id="duplicate-test" to demonstrate the error handling:
      


      

        

          
‚úÖ First Editor (valid)

          
<ace-editor id="duplicate-test" lang="javascript">
  console.log("First editor");
</ace-editor>

           console.log("First editor"); 
        

        

          
‚ùå Second Editor (duplicate ID)

          
<ace-editor id="duplicate-test" lang="javascript">
  console.log("This should fail");
</ace-editor>

           console.log("This should fail"); 
        

      


      
        
    
          
  • 
            No ID provided ‚Üí Component generates unique ID like ace-editor-1,
            ace-editor-2, etc.
          
    • 
          
  • 
            Unique ID provided ‚Üí Component validates it's unique across all ace-editors and uses it
          
    • 
          
  • 
            Duplicate ID detected ‚Üí Component throws error and shows red error box instead of loading
          
    • 
          
  • Component removed ‚Üí ID is unregistered and can be reused on a new editor

Available Attributes

      
The <ace-editor> component supports the following attributes:


      

        
          

            Attribute
            Type
            Default
            Description
          

        
        
          

            id
            String
            
              ace-editor-N
              

              (auto-generated)
            		
            
              Unique identifier for the editor. Auto-generated if not provided (e.g., ace-editor-1,
              ace-editor-2). Must be unique across all ace-editors on the page.
            
          

          

            lang
            String
            
              javascript
            
            
              Programming language mode for syntax highlighting. Default is
              javascript. Other examples: python, typescript, go,
              html, css, json, text (plain text, no highlighting),
              etc.
            
          

          

            theme
            String
            
              idle_fingers
            
            
              Visual theme for the editor. Default is
              idle_fingers (dark theme). Other popular options: monokai, github,
              dracula, solarized_dark, solarized_light, ambiance,
              etc.
            
          

          

            value
            String
            
              ""
              

              (empty)
            		
            
              Initial content of the editor. Default is empty string. Can also be provided as text content between the
              opening and closing tags. Supports both attribute and inner content approaches.
            
          

          

            readonly
            Boolean
            
              false
            
            
              Makes the editor read-only (no user editing allowed). Default is
              false (editable). Presence of attribute = true, absence = false. Example:
              <ace-editor readonly>
            
          

          

            min-height-lines
            Number
            
              none
            
            
              Minimum height of the editor measured in number of lines. No minimum by default (auto-sizes to content).
              Example:
              min-height-lines="10" ensures at least 10 lines are visible.
            
          

          

            min-height-px
            Number
            
              none
            
            
              Minimum height of the editor measured in pixels. No minimum by default (auto-sizes to content). Example:
              min-height-px="300" ensures at least 300px height. Cannot be used with
              min-height-lines simultaneously.
            
          

        
      


      
        The min-height-lines and min-height-px attributes are mutually exclusive. If both are
        provided, min-height-px
        takes precedence.

Code Execution: Displaying AND Running Code


    

      üíª Dual Purpose: The <ace-editor> component is primarily designed for
      presenting code with syntax highlighting. However, in many JavaScript scenarios, we want to both
      display the code and execute it. To avoid code duplication, the
      component provides built-in execution capabilities.
    


    

      
Two Approaches to Code Execution


      

        

          
Approach 1: data-eval Attribute

          

            Add data-eval or data-eval="module" to the <ace-editor> tag.
            Code executes after the editor is fully initialized.
          

        


        

          
Approach 2: Executable <script> Tag

          

            Use <script> with executable type attribute inside the component. Code executes
            immediately when the browser parses it.
          

        

      

    


    

      
Approach 1: Using data-eval Attribute


      
Example 1.1: data-eval (Regular Script Mode)

      

        Using data-eval alone creates a regular <script> tag in the background,
        executing code in non-ESM mode (classic script context).
      


      
        
      

      
      


      

      
Example 1.2: data-eval="module" (ESM Mode)

      

        Using data-eval="module" creates a <script type="module"> tag, executing code in
        ESM mode with import/export support.
      


      
        
      

      
      


      

      
Example 1.3: Interacting with the Editor Instance

      

        With data-eval, code executes after the editor is fully initialized, allowing you
        to interact with the editor instance programmatically.

        Instead of using data-eval, you can use <script> tags with executable type
        attributes. These execute immediately when the browser's HTML parser encounters them.
